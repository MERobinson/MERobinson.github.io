Intro to Bioinformatics

1.4 Initial quality control

The first step once you have your FASTQ data is to check the quality of the reads. For this we’ll simply run fastqc - the standard tool for this process. It’s also super easy to run and produces a html summary per sample.

1.4.1 Running FastQC

Make a new directory to store all qc info:

mkdir qc

Run FASTQC on each pair of FASTQ files:

fastqc --outdir qc fastq/?? fastq/??

For each input FASTQ, fastqc will produce a .zip archive and and .html you can open this html with a web browser.

1.4.2 Summarising with MultiQC

You can easily summarise this data accross all samples with MultiQC - just call multiqc on the qc directory:

multiqc qc

MultiQC will find any FastQC files and summarise them in interactive plots for you (it also handles a lot of other QC tool outputs too!). It should produce multiqc_report.html - open this in a web browser.

1.4.3 Interpreting Results

The report contains some general stats at the top - number of sequences etc., then specific plots for various metrics.

Some general points:

  • FastQC attempts to assign a pass/warn/fail for each metric - corresponding to the green/orange/red colouring in plots. * Some plots also have the range of values in the y axis coloured - green = high quality, orange = medium, red = poor.
  • You can hover your mouse over a specific line to find out what sample it corresponds to
  • The bar next to the plot title indicates the total number passing/warnings/failed for each metric - hover over it to see which samples are in each category
  • Plots and values can be saved through the toolbox on the right
  • click the FastQC help link to see details about each metric and possible reasons for failure

Common issues:

  • Sequence quality degrades with increasing read length so you’ll often see a drop towards the end of reads - particularly for longer reads
  • A temporary drop in quality in the middle of reads could indicate a run issue such as bubbles running through the flow cell
  • Over-represented sequences - such as contaminants or spike-ins can bias base composition, GC content and duplication
  • Base composition bias in the first ~12bp - this is often observed due to the fact that random hexamer priming isn’t really random, but doesn’t appear to have an effect downstream and can generally just be ignored.
