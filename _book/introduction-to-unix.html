3 Introduction to UNIX

You actually don’t need to know barely anything about UNIX to run a simple analysis - only how to open a terminal, input commands and 3 basic commands. This section will run through these basics, if you want a more in-depth introduction see the section on shell scripting.

3.1 The Terminal

The terminal (AKA console or shell) is an environment where you can type commands that will be interpreted and run by your system. The terminal is on all unix systems by default so just search for ‘Terminal’ in your applications to open it. It should look something like this:

The text (usually your username and current directory) followed by a dollar symbol $ is called the prompt. The prompt starts each command line and indicates your terminal is ready to accept a command - if it’s not showing up it probably means your system is still processing the previous command.

To run a process you simply type the command and press enter. Previous commands are stored and can be retrieved with the arrow buttons.

3.2 The filesystem

Files are stored on unix in a tree-like structure of directories/folders.

The base of this tree is the root directory - represented by a forward slash /. The root directory contains all other directories and files.

The location of any file is called it’s path and can be given by a series of directory names and seperated by forward slashes. e.g. the path of the file moo.txt in the filesystem diagrammed above would be /home/mark/moo.txt. An absolute path five the full path from the root directory, while a relative path gives the path relative to the current directory.

3.3 Navigating the filesystem

It’s important to remember that the console has a location - you’re always in a directory of your filesystem. By default when you start the terminal you’ll be in your home directory - /home/username.

You can print the directory your currently working in termed present working directory with the command:

pwd

To list files and directories in the current working directory, use the list command:

ls

To move to a different directory use the change directory command followed by the path to the new location. E.g. in the filesystem diagrammed above you could move to the home directory by typing:

cd /home

You can also use relative paths - if you were in home and wanted to move to Documents just type:

cd Documents

There are a number of symbols with special meanings:

  • / means root directory
  • ~ means the users home directory
  • . means current directory
  • .. means directory up from the current one

So to change from Documents to bin you could type either:

cd ../bin

or

cd ~/bin
